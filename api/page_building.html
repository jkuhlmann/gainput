Building

Gainput is built using Waf which makes it rather easy to build the library.

Simply run these commands:

  1. waf configure
  2. waf build_CONFIGNAME

There are two configurations of which you choose one by substituting CONFIGNAME for debug or release . The debug configuration supports debugging while the release configuration is more optimized.

Building Gainput as shown above, will build a dynamic-link library, a static-link library, and all samples. The executables can be found in build/CONFIGNAME/ folder.

Android NDK

In order to cross-compile for Android, the build has to be configured differently.

Naturally, the Android NDK must be installed. Replace ANDROID_NDK_PATH with the complete absolute path to your installation.

  1. waf configure --cross-android --cross-android-ndk=ANDROID_NDK_PATH
  2. waf build_CONFIGNAME

Executing these commands will also yield both a dynamic and static library in the build/CONFIGNAME/ folder.

The samples for Android need to be treated a little more to be deployed to a device even though there are some provisions present. For the time being, this is a manual process. If, for example, you want to check out the basic sample, build Gainput as described above and then do:

  1. cp build/debug/samples/basic/libbasicsample.so samples/android/libs/armeabi/libgainputsample.so
  2. cd samples/android/
  3. ANDROID_SDK_PATH/tools/android update project -p . -s --name GainputSample --target android-17
  4. ant debug
  5. ANDROID_SDK_PATH/platform-tools/adb install bin/GainputSample-debug.apk

Where ANDROID_SDK_PATH is the path to your Android SDK installation.
